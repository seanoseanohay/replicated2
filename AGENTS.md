# Project agent instructions

## Project rules

-   Read `README.md` before major changes.
-   Read documentation only when relevant to the task.
-   Check `docs/decisions.md` before proposing architecture changes.

## Engineering rules

-   Follow the existing project structure and conventions.
-   Keep changes scoped and easy to review.
-   Do not rewrite working code without a clear reason.
-   Prefer extending existing patterns over introducing new frameworks
    or abstractions.

## Documentation rules

-   Update `docs/scope.md` if scope changes.
-   Update `docs/phases.md` when phases complete or materially change.
-   Update `docs/architecture.md` when system structure changes.
-   Record major technical decisions in `docs/decisions.md`.

## Delivery rules

-   Work in phases when possible.
-   Finish one coherent slice before moving forward.
-   Explain what changed, how it was verified, and any remaining risks.
